Build CommitGenerator orchestration and MessageValidator quality gates — the core intelligence that wires detectors to AI generation and enforces quality requirements.

MessageValidator enforces QUAL-01 and QUAL-02: rejects generic messages like "update code" or "fix bug" (<5% allowed), requires reasoning/why sections in >90% of messages, and validates component scope when detected.

Purpose: End-to-end commit generation with quality gates ensuring specific, complete messages.

    - Given file types detected, prompt includes "File types: 3 JS, 2 JSON, 1 MD"
    - Given AI messages returned, passes through validator before returning

    - Given error during generation, logs to activity logger and throws
  </behavior>
  <action>
    Create `src/core/commit-generator.js` as a class `CommitGenerator`:

    1. **Constructor:** Accept dependencies object:
       ```javascript
       {
         gitManager, configManager, analysisEngine, messageFormatter,
         providerOrchestrator, activityLogger, statsManager,
         ComponentDetector, FileTypeDetector, DependencyMapper
       }
       ```
       Store all as instance properties.

    2. **`generate(options)`** — Main orchestration method. Accepts options object with:
       - `count` (number, default 3)
       - `type` (string, optional conventional type)
       - `conventional` (boolean, default true)
       - `language` (string, default 'en')
       - `dryRun` (boolean, default false)
       - `cache` (boolean, default true)

       Returns Promise<messages[]> or throws on validation failure.

       **Pipeline steps:**
       a. Get staged diff via `gitManager.getStagedDiff()`
       b. If no diff, throw "No staged changes"
       c. Extract changed file paths from diff
       d. Run detectors in parallel:
          - `componentDetector.detect(filePaths)`
          - `fileTypeDetector.detectBatch(filePaths)`
          - `dependencyMapper.mapDependencies(filesWithContents)`
       e. Build enriched context object:
          ```javascript
          {
            components: { list, summary },
            fileTypes: { summary },
            dependencies: { imports, exports, affected },
            semantic: analysisEngine.analyzeRepository()
          }
          ```
       f. Call `providerOrchestrator.generateWithSequentialFallback(diff, { context: enrichedContext, ...options })`
       g. Pass messages through `MessageValidator.validate(messages, context)`
       h. If validation fails, throw with suggestions
       i. Format messages via `messageFormatter.formatWithContext(msg, context, options)`
       j. If not dryRun, commit via `gitManager.commit(selectedMessage)`
       k. Return formatted messages

    3. **`extractFilePathsFromDiff(diff)`** — Helper method. Parse diff --git lines to extract file paths. Returns string[].

    4. **`buildEnrichedPrompt(diff, detectorResults)`** — Build AI prompt that includes:
       - Original diff
       - "Changed components: X, Y, Z"
       - "File types: 5 JavaScript, 2 JSON, 1 Markdown"
       - "Dependencies: 3 files affected by these changes"
       - "Semantic analysis: [key changes from analysisEngine]"

    Use `require('path')`. CommonJS `module.exports = CommitGenerator`.

    Per CORE-01: Full git workflow support (stage, commit, pull, push).
    Per CORE-02: AI provider abstraction with sequential fallback.

<task type="auto" tdd="true">
  <name>Task 2: Create MessageValidator (QUAL-01, QUAL-02)</name>
  <files>src/core/message-validator.js, tests/core/message-validator.test.js</files>
  <behavior>
    - Given message "update code", validate() returns { valid: false, reason: "generic" }
    - Given message "fix bug in auth module", returns { valid: false, reason: "generic" }
    - Given message "fix: resolve race condition in AuthService.login()", returns { valid: true }
    - Given message without reasoning (no why section), returns { valid: false, reason: "no-reasoning" }
    - Given message with component scope when component detected, returns { valid: true }
    - Given batch of messages, validateBatch() returns { validMessages, invalidMessages, stats }
    - Given <5% valid messages, throws QualityError with suggestions
    - Given >90% messages with reasoning, passes QUAL-02 check
  </behavior>
  <action>
    Create `src/core/message-validator.js` as a class `MessageValidator`:

    1. **Constructor:** No dependencies needed.

    2. **`validate(message, context)`** — Main validation method. Accepts message string and optional context object. Returns:
       ```javascript
       {
         valid: boolean,
         score: number (0-100),
         issues: string[],
         suggestions: string[]
       }
       ```

       **Validation rules:**
       - **Generic check (QUAL-01):** Reject if message matches generic patterns:
         - "update code", "fix bug", "changes", "misc updates", "various fixes"
         - Pattern: `/\b(update|change|modify|fix|add)\s+(code|bug|stuff|things|features?)\b/i`
         - Pattern: `/^(update|fix|change|misc|various)\s*$/i`
         - Pattern: `/^(improvements?|bug fix|updates?|refactor)\s*$/i`

       - **Specificity check:** Reward specific technical terms:
         - Function/class names: `/\b[A-Z][a-zA-Z]*\b/`
         - File paths: `/\b[\w\-\/]+\.(js|ts|py|java|json)\b/`
         - Actions with objects: `/\b(add|create|remove|fix)\s+\w+\b/i`

       - **Reasoning check (QUAL-02):** Require why/reasoning:
         - Check for "because", "to", "for", "so that", "enables", "allows"
         - Check for problem statement: "fixes", "resolves", "prevents"
         - If no reasoning detected → issue: "no-reasoning"

       - **Component scope check:** If context.components detected, require scope in message:
         - Check conventional format: `type(component): description`
         - If component detected but not in message → warning (not rejection)

       **Scoring:**
       - Start at 50
       - +20 for conventional format
       - +15 for specific technical terms
       - +15 for reasoning detected
       - +10 for component scope match
       - -30 for generic patterns
       - -50 for banned patterns (single word: "update", "fix", "changes")

    3. **`validateBatch(messages, context)`** — Validate array of messages. Returns:
       ```javascript
       {
         validMessages: Array<{ message, score, issues }>,
         invalidMessages: Array<{ message, score, issues, suggestions }>,
         stats: {
           total: number,
           validCount: number,
           genericCount: number,
           withReasoning: number,
           qualityRate: number (0-1)
         }
       }
       ```

    4. **`checkQualityThresholds(batchResult)`** — Check QUAL-01 and QUAL-02:
       - QUAL-01: `genericCount / total < 0.05` (<5% generic)
       - QUAL-02: `withReasoning / total > 0.90` (>90% with reasoning)
       - Returns { qual01Pass, qual02Pass, failures }

    5. **`generateSuggestions(issues)`** — Return helpful suggestions:
       - If generic: "Be more specific: mention function/class names changed"
       - If no-reasoning: "Add why: 'to fix X bug', 'enables Y feature'"
       - If no-scope: "Consider adding scope: 'feat(auth): add login validation'"

    Use CommonJS `module.exports = MessageValidator`.

    Per QUAL-01: <5% generic messages allowed.
    Per QUAL-02: >90% messages include reasoning.
    Per D-06: Return flat context objects.
  </action>
