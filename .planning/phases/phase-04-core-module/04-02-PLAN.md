<objective>
Build GitWorkflow and AutoWorkflow — the git automation layer that provides full git workflow support and intelligent auto-git capabilities.

GitWorkflow orchestrates git operations: stage changes (selective or all), commit with messages, pull from remote with merge/rebase, and push to remote branch. It wraps git-manager with higher-level workflow logic.

AutoWorkflow provides the "one command" automation: stage → commit → pull → resolve conflicts → push. It uses AI-powered conflict resolution when merge conflicts are detected, with fallback to manual resolution options.

Purpose: Complete git automation with intelligent conflict handling.
Output: Two tested workflow modules with full git lifecycle support.
</objective>

  <files>src/core/git-workflow.js, tests/core/git-workflow.test.js</files>
  <behavior>
    - Given files to stage, stage() adds them to git staging area
    - Given "all" mode, stageAll() stages all changed files
    - Given commit message, commit() creates git commit with message
    - Given pull needed, pullWithStrategy() uses merge or rebase per config
    - Given push after commit, push() pushes to remote branch
    - Given divergent branches, detects and warns before push
    - Given error during git op, logs to activity logger and throws descriptive error
  </behavior>
  <action>
    Create `src/core/git-workflow.js` as a class `GitWorkflow`:

    1. **Constructor:** Accept dependencies object:
       ```javascript
       {
         gitManager, configManager, activityLogger
       }
       ```
       Store all as instance properties.

    2. **`stage(files, options)`** — Stage files for commit. Accepts:
       - `files`: string[] (file paths) or 'all'
       - `options`: { dryRun?: boolean }

       If files === 'all', calls `gitManager.stageAll()`.
       Otherwise, calls `gitManager.stageFiles(files)`.

       Returns { stagedFiles: string[], count: number }.

    3. **`commit(message, options)`** — Create commit. Accepts:
       - `message`: string (commit message)
       - `options`: { dryRun?: boolean, sign?: boolean }

       Validates message is not empty.
       Calls `gitManager.commit(message)`.

       Returns { success: true, hash: string }.

    4. **`pullWithStrategy(options)`** — Pull from remote. Accepts:
       - `options`: { strategy?: 'merge'|'rebase', dryRun?: boolean }

       Checks if pull is needed (divergent or behind).
       If strategy === 'rebase', uses git rebase.
       Otherwise, uses git merge.

       Returns { pulled: boolean, strategy: string, changes: number }.

    5. **`push(options)`** — Push to remote. Accepts:
       - `options`: { dryRun?: boolean, force?: boolean }

       Checks if commit exists to push.
       Checks for divergent branches.
       Calls `gitManager.push(force)`.

       Returns { success: true, remote: string, branch: string }.

    6. **`getStatus()`** — Get git status. Returns:
       ```javascript
       {
         staged: string[],
         unstaged: string[],
         untracked: string[],
         ahead: number,
         behind: number,
         hasConflicts: boolean
       }
       ```

    7. **`validateState()`** — Validate git state before workflow. Returns:
       - Checks: is repo, has staged changes, no conflicts, remote exists
       - Returns { valid: boolean, issues: string[] }

    Use `require('simple-git')` if needed for low-level ops. CommonJS `module.exports = GitWorkflow`.

    Per CORE-01: Full git workflow support.
    Per AUTO-01: Automatic staging support.
    Per AUTO-02: Pull-before-push integration.

<task type="auto" tdd="true">
  <name>Task 2: Create AutoWorkflow (AUTO-01, AUTO-02, AUTO-03, AUTO-04)</name>
  <files>src/core/auto-workflow.js, tests/core/auto-workflow.test.js</files>
  <behavior>
    - Given auto() call, runs full pipeline: stage → commit → pull → push
    - Given conflicts detected during pull, offers AI resolution
    - Given AI resolution success, applies merged result and continues
    - Given AI resolution failure, offers manual options (theirs/mine/manual/cancel)
    - Given --skip-pull option, skips pull step
    - Given --no-push option, skips push step
    - Given dry-run mode, simulates without making changes
    - Given error at any step, rolls back and provides helpful error message
  </behavior>
  <action>
    Create `src/core/auto-workflow.js` as a class `AutoWorkflow`:

    1. **Constructor:** Accept dependencies object:
       ```javascript
       {
         gitWorkflow, conflictResolver, commitGenerator,
         configManager, activityLogger, statsManager
       }
       ```
       Store all as instance properties.

    2. **`auto(options)`** — Main auto-git method. Accepts:
       - `skipPull`: boolean (default false)
       - `noPush`: boolean (default false)
       - `dryRun`: boolean (default false)
       - `conflictStrategy`: 'ai'|'manual'|'theirs'|'mine' (default 'ai')

       **Pipeline:**
       a. Log workflow start with session ID
       b. Validate git state via `gitWorkflow.validateState()`
       c. Stage all changes via `gitWorkflow.stage('all')`
       d. If no staged changes, exit with "nothing to commit"
       e. Generate commit message via `commitGenerator.generate({ dryRun })`
       f. If not dryRun, create commit via `gitWorkflow.commit(message)`
       g. If skipPull is false:
          - Check if pull needed via `gitWorkflow.getStatus()`
          - If behind remote, call `pullWithConflictHandling(options)`
       h. If noPush is false and not dryRun, push via `gitWorkflow.push()`
       i. Log workflow completion
       j. Return { success: true, commits: 1, pushed: boolean, pulled: boolean }

    3. **`pullWithConflictHandling(options)`** — Pull with conflict resolution. Accepts options. Returns:
       - Tries `gitWorkflow.pullWithStrategy()`
       - If conflicts detected:
         - If conflictStrategy === 'ai': call `resolveConflictsWithAI()`
         - If conflictStrategy === 'manual': call `resolveConflictsManual()`
         - If conflictStrategy === 'theirs': accept incoming changes
         - If conflictStrategy === 'mine': keep current changes
       - Retries pull after resolution
       - Returns { pulled: true, conflictsResolved: number }

    4. **`resolveConflictsWithAI(conflictFiles, options)`** — AI-powered resolution. Accepts:
       - `conflictFiles`: string[] (files with conflicts)
       - `options`: { language?: string }

       For each conflict file:
       a. Read file content with conflict markers
       b. Parse conflict blocks via `conflictResolver.parseConflictBlocks()`
       c. For each block, call `conflictResolver.resolveConflictWithAI()`
       d. Write resolved content back to file
       e. Stage resolved file

       Returns { resolved: number, failed: number }.

    5. **`resolveConflictsManual(conflictFiles)`** — Manual resolution guidance. Returns:
       - Lists conflict files
       - Provides instructions: "Run 'git mergetool' or edit files manually"
       - Waits for user to resolve
       - Returns { resolved: number }

    6. **`rollback(lastGoodState)`** — Rollback on failure. Accepts last good state. Performs:
       - If commit created but push failed: git reset --soft HEAD~1
       - If files staged but commit failed: git reset
       - Logs rollback actions

    7. **`provideErrorSuggestions(error, step)`** — Helpful error messages:
       - If auth error: "Check remote authentication credentials"
       - If conflict error: "Use --conflict-strategy=ai for auto resolution"
       - If network error: "Check internet connection and retry"
       - If permission error: "Check file permissions and try again"

    Use `require('fs-extra')` for file operations. CommonJS `module.exports = AutoWorkflow`.

    Per AUTO-01: Automatic staging.
    Per AUTO-02: Pull-before-push.
    Per AUTO-03: AI conflict resolution.
    Per AUTO-04: Automatic push.
  </action>
