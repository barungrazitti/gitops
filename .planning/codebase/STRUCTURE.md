# Codebase Structure

**Analysis Date:** 2026-03-26

## Directory Layout

```
gitops/
├── bin/                    # CLI entry points
```

## Directory Purposes

**bin/:**
- Purpose: Executable CLI entry points
- Contains: Commander.js CLI definitions
- Key files: `aic.js` (auto workflow), `aicommit.js` (commit generation only)

**src/core/:**
- Purpose: Domain services and business logic
- Contains: Git operations, configuration, caching, analysis, logging, stats
- Key files:
  - `git-manager.js` - All git operations (diff, commit, status, history)
  - `config-manager.js` - Configuration management with Joi validation
  - `cache-manager.js` - Semantic caching with similarity detection
  - `analysis-engine.js` - Repository context analysis (semantic, WordPress detection)
  - `message-formatter.js` - Conventional commit formatting and validation
  - `stats-manager.js` - Usage statistics tracking
  - `activity-logger.js` - Comprehensive activity logging
  - `hook-manager.js` - Git hooks installation/management
  - `circuit-breaker.js` - Circuit breaker pattern for AI services

**src/providers/:**
- Purpose: AI service abstraction layer
- Contains: Base provider, concrete implementations, factory
- Key files:
  - `base-provider.js` - Abstract base class with common provider logic
  - `groq-provider.js` - Groq cloud API implementation
  - `ollama-provider.js` - Ollama local API implementation
  - `ai-provider-factory.js` - Provider instantiation and metadata

**src/utils/:**
- Purpose: Shared utility functions and helpers
- Contains: Prompt builders, diff processors, validators, performance tools
- Key files:
  - `efficient-prompt-builder.js` - Optimized prompt generation
  - `optimized-diff-processor.js` - Diff preprocessing and chunking
  - `secret-scanner.js` - PII and secret detection/redaction
  - `input-sanitizer.js` - Input validation and sanitization
  - `performance-utils.js` - Token estimation and performance tracking
  - `project-type-detector.js` - Project type detection (WordPress, monorepo, etc.)
  - `retry-utility.js` - Retry logic with exponential backoff
  - `fallback-commit-generator.js` - Fallback message generation

**tests/:**
- Purpose: Jest test suites
- Contains: Unit tests for all modules, mocks
- Key files:
  - `setup.js` - Test configuration and global mocks
  - `aicommit-cli.test.js` - CLI integration tests
  - `git-manager.test.js` - Git operations tests
  - `groq-provider.test.js` - Groq provider tests
  - `ollama-provider.test.js` - Ollama provider tests
  - `cache-manager.test.js` - Caching tests
  - `activity-logger.test.js` - Logging tests
  - `mocks/` - Mock implementations

**.planning/:**
- Purpose: Project planning and codebase analysis documents
- Contains: Architecture docs, structure docs, concerns
- Generated by: GSD mapping commands

## Naming Conventions

**Files:**
- Classes: PascalCase (e.g., `AICommitGenerator`, `GitManager`, `ConfigManager`)
- Utilities: kebab-case (e.g., `efficient-prompt-builder.js`, `secret-scanner.js`)
- Tests: `*.test.js` suffix (e.g., `git-manager.test.js`)
- Binaries: kebab-case (e.g., `aic.js`, `aicommit.js`)

**Classes:**
- Always PascalCase
- Descriptive names indicating responsibility
- Modules export single class typically

**Functions/Methods:**
- camelCase (e.g., `generateCommitMessages`, `analyzeRepository`)
- Async functions: prefixed with `async`
- Private methods: prefixed with `_` (convention, not enforced)

**Constants:**
- UPPER_SNAKE_CASE (e.g., `MAX_TOKEN_LIMIT`, `DEFAULT_TIMEOUT`)

## Where to Add New Code

**New Feature:**
- Primary code: `src/core/[feature-name].js`
- Tests: `tests/[feature-name].test.js`
- If feature is AI-related: `src/providers/[provider-name]-provider.js`
- If feature is utility: `src/utils/[utility-name].js`

**New Component/Module:**
- Implementation: `src/core/` for business logic, `src/utils/` for helpers
- Class file: PascalCase filename matching class name
- Export: `module.exports = ClassName;`

**Utilities:**
- Shared helpers: `src/utils/[utility-name].js`
- Kept framework-agnostic (no business logic)
- Exported as functions or classes as appropriate

**New AI Provider:**
- Implementation: `src/providers/[provider]-provider.js`
- Extend: `BaseProvider` class
- Register: Add to `AIProviderFactory.create()` switch statement
- Tests: `tests/[provider]-provider.test.js`

**Configuration Options:**
- Add to: `src/core/config-manager.js` (getDefaults schema)
- Add validation: Joi schema in getValidationSchema()
- Document: Update README with new options

## Special Directories

**.aic-logs/:**
- Purpose: Runtime activity logs
- Generated: Yes, at runtime
- Committed: No (in .gitignore)
- Content: JSON log entries with timestamps, actions, system info
- Retention: 30 days, auto-cleanup

**coverage/:**
- Purpose: Jest test coverage reports
- Generated: Yes, by `npm run test:coverage`
- Committed: No (in .gitignore)
- Content: HTML coverage reports, lcov reports

**node_modules/:**
- Purpose: npm dependencies
- Generated: Yes, by `npm install`
- Committed: No (in .gitignore)

**.planning/codebase/:**
- Purpose: Codebase analysis documentation
- Generated: Yes, by GSD mapping commands
- Committed: Yes
- Content: ARCHITECTURE.md, STRUCTURE.md, CONCERNS.md, etc.

**scripts/:**
- Purpose: Build and deployment automation
- Generated: No (committed)
- Content: Shell scripts for deployment, building
