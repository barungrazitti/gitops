  <action>
    Create .prettierrc file in project root with the following configuration:
    
    {
      "singleQuote": true,
      "trailingComma": "es5",
      "tabWidth": 2,
      "semi": true,
      "printWidth": 80,
      "endOfLine": "lf"
    }
    
    Configuration details:
    - singleQuote: true - Use single quotes instead of double quotes
    - trailingComma: "es5" - Add trailing commas where valid in ES5 (objects, arrays)
    - tabWidth: 2 - Use 2 spaces for indentation
    - semi: true - Add semicolons at the end of statements
    - printWidth: 80 - Line length limit
    - endOfLine: "lf" - Line ending style (consistent across platforms)
  </action>
  <acceptance_criteria>

    - .prettierrc contains "semi": true

## Verification Criteria

1. Running `npm run format` formats all JavaScript files in src/, tests/, bin/, and scripts/
2. Running `npm run format:check` verifies formatting without making changes
3. Prettier uses single quotes, trailing commas (ES5), 2-space indentation, and semicolons
4. ESLint extends order includes "prettier" last to disable conflicting formatting rules

## Must-Haves

- Prettier must be configured with single quotes, trailing commas (ES5), and 2-space indentation
- Prettier must run manually via npm script only (no pre-commit hooks or on-save automation)
- eslint-config-prettier must be used to disable ESLint rules that conflict with Prettier
- Prettier must format all JavaScript files in src/, tests/, bin/, and scripts/ directories
