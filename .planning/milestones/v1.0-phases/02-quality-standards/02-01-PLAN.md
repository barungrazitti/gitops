## Objective

Install and configure ESLint with eslint-config-airbnb-base to enforce consistent JavaScript code style across the entire project, with strict rule enforcement from day one.

## Tasks

<task>
  <read_first>
    - package.json
  </read_first>
  <action>
    Install ESLint dependencies using npm:
    
    npm install --save-dev eslint eslint-config-airbnb-base eslint-config-prettier eslint-plugin-import
    
    This installs:
    - eslint: Core ESLint linter
    - eslint-config-airbnb-base: Airbnb style guide configuration (without React)
    - eslint-config-prettier: Disables ESLint rules that conflict with Prettier
    - eslint-plugin-import: Required peer dependency for airbnb-base
  </action>
  <acceptance_criteria>
    - package.json devDependencies contains "eslint"
    - package.json devDependencies contains "eslint-config-airbnb-base"
    - package.json devDependencies contains "eslint-config-prettier"
    - package.json devDependencies contains "eslint-plugin-import"
  </acceptance_criteria>
</task>

<task>
  <read_first>

<task>
  <read_first>
    - package.json
  </read_first>
  <action>
    Create .eslintrc.json file in project root with the following configuration:
    
    {
      "extends": ["airbnb-base", "prettier"],
      "env": {
        "node": true,
        "jest": true,
        "es2021": true
      },
      "parserOptions": {
        "ecmaVersion": 2021,
        "sourceType": "script"
      },
      "rules": {
        "no-console": "warn",
        "no-unused-vars": ["error", { "argsIgnorePattern": "^_" }],
        "no-undef": "error",
        "import/no-extraneous-dependencies": ["error", { "devDependencies": true }]
      },
      "ignorePatterns": ["node_modules/", "coverage/", "*.log"]
    }
    
    Configuration details:
    - extends: airbnb-base for Airbnb style, prettier to disable conflicting rules
    - env: node for CommonJS, jest for test globals, es2021 for modern features
    - parserOptions: sourceType "script" for CommonJS (not modules)
    - rules: no-console as warn (catch console.log statements), no-unused-vars with _ pattern for intentional unused params, no-undef to catch undeclared variables
  </action>
  <acceptance_criteria>
    - File .eslintrc.json exists in project root
    - .eslintrc.json contains "extends": ["airbnb-base", "prettier"]
    - .eslintrc.json contains "env": { "node": true, "jest": true, "es2021": true }
    - .eslintrc.json contains "parserOptions": { "ecmaVersion": 2021, "sourceType": "script" }
    - .eslintrc.json contains "no-console": "warn"
    - .eslintrc.json contains "no-unused-vars" rule with argsIgnorePattern
    - .eslintrc.json contains "no-undef": "error"
    - .eslintrc.json contains "ignorePatterns" array
  </acceptance_criteria>
</task>

## Verification Criteria

1. Running `npm run lint` executes ESLint on src/, tests/, bin/, and scripts/ directories
2. ESLint configuration includes Airbnb base rules with Prettier compatibility
3. Configuration supports CommonJS require() syntax and Jest test environment
4. ESLint can detect undeclared variables, unused imports, and console.log statements

## Must-Haves

- ESLint must use eslint-config-airbnb-base as-is with no custom rule overrides
- All Airbnb rules must be enforced as errors (strict from day one)
- ESLint must lint all JavaScript files in src/, tests/, bin/, and scripts/ directories
- Configuration must support CommonJS project (sourceType: "script")
- Jest globals must be available in test files (env: { jest: true })
